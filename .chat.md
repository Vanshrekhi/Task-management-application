# Taskify — Real-time chat (Socket.IO)

## Overview

Chat uses **Socket.IO** on the same origin as the API (HTTP server + `Server` from `socket.io`). Clients authenticate with the **JWT cookie** (`token`). Rooms are identified by a **string key** stored in MongoDB (`ChatRoom`). Messages are **persisted** on the room document and **broadcast** to everyone in the Socket.IO room `room:<key>`.

Backend registration lives in `server/socket/chat.js` (loaded from `server/index.js`). The client UI is `client/src/pages/ChatRoom.jsx`.

---

## Connection

1. Browser opens Socket.IO connection (with credentials so cookies are sent).
2. Server middleware verifies JWT from `Cookie: token=...` and loads `User` (`_id`, `name`, `email`).
3. Socket joins `user:<userId>` for direct notifications (e.g. join requests to host).

---

## Flow (happy path)

1. **Host** creates a room via REST (`POST /api/chat/rooms` — see `chatRoute` / `chatController`) → DB row with `key`, `host`, `teamMembers`, `messages`, `isActive`.
2. **Host** connects socket and emits **`chat:join-room`** / **`join-room`** with `{ key }` → joins Socket.IO room `room:<key>`, receives **`chat:history`** (and **`receive-message`** with `type: "history"`).
3. **Member** uses **`chat:request-join`** with `{ key }` → pending membership; host gets **`chat:join-request`** on `user:<hostId>`; member gets **`chat:join-pending`**.
4. Host emits **`chat:respond-request`** `{ key, memberId, approve }` → member receives **`chat:join-response`** `{ key, approved }`.
5. On approval, member emits **`chat:join-room`** / **`join-room`** → can send messages.
6. **`chat:send-message`** / **`send-message`** `{ key, text }` → message saved in DB → broadcast **`chat:new-message`** and **`receive-message`** (`type: "message"`) to `room:<key>`.
7. Host may **`chat:end-session`** / end flow → room `isActive: false`, messages cleared → **`chat:session-ended`** + **`receive-message`** (`type: "session-ended"`).

---

## Socket events (server listens)

| Event | Payload | Purpose |
|--------|---------|--------|
| `chat:request-join` | `{ key }` | Ask to join a room (may create pending membership) |
| `chat:respond-request` | `{ key, memberId, approve }` | Host approves/rejects join |
| `chat:join-room` | `{ key }` | Enter room + load history (alias: **`join-room**`) |
| `chat:send-message` | `{ key, text }` | Persist + broadcast (alias: **`send-message**`) |
| `chat:end-session` | `{ key }` | Host ends room |
| `chat:leave-room` | `{ key }` | Leave Socket.IO room (alias: **`leave-room**`) |

---

## Socket events (server emits to client)

| Event | When |
|--------|------|
| `chat:error` | Invalid key / not accepted / inactive room |
| `chat:join-approved` | Join OK (host or accepted member) |
| `chat:join-rejected` | Previously rejected |
| `chat:join-pending` | Waiting for host |
| `chat:join-request` | Host: someone wants to join |
| `chat:join-response` | Member: approve/reject result |
| `chat:history` | `{ key, messages }` after join |
| `chat:new-message` | `{ key, message }` (legacy name) |
| `receive-message` | Same payload + `type`: `"history"` \| `"message"` \| `"session-ended"` |
| `chat:session-ended` | `{ key }` |
| `left-room` | Client ack after **`leave-room**` |

## Example payloads

**Join room**

```json
{ "key": "ROOM-ABC123" }
```

**Send message**

```json
{ "key": "ROOM-ABC123", "text": "Hello team" }
```

**Broadcast (`chat:new-message` / `receive-message`)**

```json
{
  "key": "ROOM-ABC123",
  "message": {
    "sender": { "_id": "...", "name": "Ada" },
    "text": "Hello team",
    "sentAt": "2026-04-20T12:00:00.000Z"
  }
}
```

## Frontend notes

- Set `VITE_SOCKET_URL` to your API origin in production if the client is served from a different host.
- Vite dev server should **proxy** `/socket.io` to the API (see `client/vite.config.js`).
